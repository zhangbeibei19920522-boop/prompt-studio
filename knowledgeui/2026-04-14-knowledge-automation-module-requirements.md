# OPPO 知识库自动化产品模块需求简述

## 1. 背景

当前 `/Users/cs001/oppo_support` 已经有一套本地离线知识库构建链路，覆盖源文件盘点、原始记录抽取、数据清洗、分流、高风险识别、结构化、同题合并、冲突检测、发布门禁、parents/chunks 生成和检索索引产物生成。

线上检索服务在 `/Users/cs001/oppo_support/monorepo/retrieve`，已经支持快照入库、任务状态查询和召回接口。检索试查能力可复用已有 retrieve recall 能力，作为运营发布前的人工 spot check。

本需求的目标不是重做清洗和检索逻辑，而是把已有清洗脚本、索引生成和 retrieve 入库能力做成 cqil-studio 中可操作、可追踪、可迭代的线上产品模块。

## 2. 已确认边界

- 页面位置：放在 `cqil-studio` 的知识库自动化新页面。
- 层级关系：客户是最高层级，一个客户只有一个知识库。页面只切换客户，不再提供知识库下拉切换。
- 首次建库：客户下可以没有知识库。无知识库时首页展示空状态和 `创建知识库` 按钮，进入第一版全量构建流程。
- 维护任务：客户已有知识库后，运营通过 `新建维护任务` 做批量文件更新、人工补充、内容修复或全量重建。
- 人工审核：审核结论只对当前任务和当前轮次生效，不沉淀为全局清洗规则。
- 外链抓取：第一版不支持 TXT 外链内容抓取，只处理上传的本地文件和人工录入内容。
- 发布策略：每次清洗轮次生成独立的内部版本。正式发布时只切换当前客户知识库指向的线上版本，历史版本不删除，可查看和回滚。
- 用户侧措辞：页面不直接暴露 `source_id`、`base_source_id`、`upload_mode` 等工程字段。文档中如需描述，统一标记为内部字段。
- 测试范围：第一版不做自动化测试集、测试覆盖率页面和自动化测试报告页面。质量确认收敛为风险确认、清洗结果确认、检索试查和发布门禁。

## 3. 运营场景

### 3.1 创建知识库

适用于客户第一次接入，当前客户还没有知识库。

运营流程：

1. 在顶部选择客户。
2. 首页看到“当前客户还没有知识库”空状态。
3. 点击 `创建知识库`。
4. 填写知识库名称和任务名称。
5. 上传第一版完整原文件。
6. 配置风险确认方式、自动优化轮数和完成提醒。
7. 启动清洗流程。
8. 在任务详情中处理风险、确认清洗结果、生成检索索引、做检索试查。
9. 无待确认项且索引已更新后发布，形成该客户的第一版线上知识库。

### 3.2 批量文件更新

适用于客户已有知识库，运营需要上传一批新文件或替换一批旧文件。

运营流程：

1. 点击 `新建维护任务`。
2. 任务类型选择 `批量文件更新`。
3. 上传本次文件批次。
4. 系统自动解析、清洗、识别风险与冲突。
5. 如果存在风险项，先进入 `风险与确认` 处理。
6. 风险处理完成后，进入 `清洗结果确认` 查看和编辑运营可读内容草稿。
7. 生成检索索引并进入 `检索试查`。
8. 试查符合预期后发布。

### 3.3 人工补充

适用于小范围新增 FAQ、政策说明或运营临时说明，不需要上传原文件。

运营流程：

1. 点击 `新建维护任务`。
2. 任务类型选择 `人工补充`。
3. 在页面中一次新增多条内容，每条包含标题、正文和来源说明。
4. 创建后直接进入 `清洗结果确认`，继续编辑内容草稿。
5. 保存后生成检索索引。
6. 到 `检索试查` 确认新增内容可被搜到。
7. 发布上线。

### 3.4 内容修复

适用于线上回答不完整、不准确或搜不到。

运营流程：

1. 点击 `新建维护任务`。
2. 任务类型选择 `内容修复`。
3. 支持上传修复后的原文件或补充文件。
4. 支持手动新增多个待修复问题，每个问题包含用户问题、当前问题描述和期望修复方向。
5. 系统按本次修复来源重新清洗并生成内容草稿。
6. 在 `检索试查` 使用待修复问题验证效果。
7. 如果不符合预期，点击 `启动下一轮优化`，填写优化方向描述。
8. 新一轮优化完成后继续试查，直到满足发布要求。

## 4. 一期产品流程

```text
选择客户
-> 无知识库：创建知识库 / 有知识库：新建维护任务
-> 上传原文件或录入人工内容
-> 自动解析与清洗
-> 风险与冲突确认
-> 清洗结果确认与人工编辑
-> 生成检索索引
-> 检索试查
-> 发布
-> 首页和发布结果页可查看当前线上版本与历史版本
```

流程规则：

- 有风险项时，不展示或不允许进入清洗结果确认，必须先处理风险与冲突。
- 有待确认项时不允许发布。
- 清洗结果被人工编辑后，索引状态变为待更新，必须重新生成检索索引后才能发布。
- 已发布上线的任务只读。除 `发布结果`、版本查看和回滚类入口外，其他处理按钮都隐藏。
- 历史发布版本仍展示在首页列表中，但只能查看，不允许继续处理、试查或再次发布。

## 5. 功能范围

### 5.1 客户与知识库状态

顶部上下文只包含：

- 当前客户选择。
- 当前客户的知识库状态。
- 当前线上版本状态。

展示规则：

- 客户无知识库：展示 `尚未创建知识库` 和 `暂无线上版本`。
- 客户有知识库但未发布：展示知识库名称和 `暂无线上版本`。
- 客户有已发布版本：展示知识库名称和当前线上版本名称。

### 5.2 首页任务列表

首页用于查看当前客户知识库的维护任务、处理状态、当前线上版本和历史发布版本。

顶部操作：

- `刷新`
- `新建维护任务`，仅客户已有知识库时展示。
- `创建知识库`，仅客户无知识库时在空状态中展示。

统计卡片：

- 正在处理。
- 待人工确认。
- 可发布任务。
- 当前线上版本。该值可点击，跳转到对应任务的发布结果页。

列表字段：

- 任务名称。
- 任务类型。
- 知识库 / 内容来源。
- 状态。
- 当前阶段。
- 待确认。
- 本轮产出。
- 最终发布版本。
- 操作。

历史版本规则：

- 首页必须展示当前线上版本和已被替换的历史发布版本。
- 当前线上版本标记为 `当前线上`。
- 被替换版本标记为 `历史版本` 或 `已被新版替换`。
- 历史版本只允许查看发布结果或查看版本内容。

### 5.3 新建流程

新建流程采用四步：

```text
基础信息 -> 内容来源 -> 运行参数 -> 确认启动
```

无知识库客户进入 `创建知识库`：

- 所属客户只读。
- 填写新知识库名称。
- 不展示维护对象。
- 不展示任务类型。
- 固定为第一版全量构建。
- 不展示历史上传批次。

已有知识库客户进入 `新建维护任务`：

- 所属客户只读。
- 当前知识库只读。
- 不展示维护对象。
- 任务类型支持：批量文件更新、人工补充、内容修复、全量重建。

内容来源规则：

- 批量文件更新：上传原文件。
- 全量重建：上传完整原文件。
- 人工补充：不上传原文件，支持一次新增多条内容。
- 内容修复：支持上传修复后的原文件，也支持手动新增待修复问题，两种方式可同时使用。

运行参数只保留运营能理解的配置：

- 风险确认方式。
- 自动优化清洗轮数。
- 完成后提醒。

不展示：

- 本次处理方式。
- 基于哪份线上内容。
- 内容生成方式。
- 召回率、测试覆盖率、测试集条数等测试相关配置。

### 5.4 任务详情

任务详情是核心工作台，包含：

- 顶部摘要：当前轮次、当前处理状态、待人工确认、下一步操作。
- 处理进度与维护记录。
- 风险与确认。
- 清洗结果确认。
- 清洗轮次记录。
- 检索试查。
- 发布结果。

处理进度节点：

```text
任务创建
-> 内容来源
-> 解析与清洗
-> 风险与冲突确认
-> 清洗结果确认
-> 检索试查
-> 发布
```

### 5.5 风险与确认

风险与确认用于处理阻断清洗结果进入下一步的事项。

Tab：

- 待处理事项。
- 合并冲突确认。
- 高风险内容删除。

合并冲突确认要求：

- 每个冲突项使用独立大卡片展示，不使用表格作为主要确认界面。
- 展示当前线上内容、本次上传内容、来源和系统建议。
- 操作包括保留当前线上内容、使用本次上传内容、本轮忽略。

高风险内容删除要求：

- 展示被删除或阻断的内容、来源、风险分类、删除原因和触发阶段。
- 操作包括保持删除、恢复到本轮内容。
- 恢复动作只对当前任务生效。

### 5.6 清洗结果确认

清洗结果确认展示运营可读的内容草稿，不展示切片、向量和底层索引结构。

页面能力：

- 查看清洗后的标题、正文和来源。
- 搜索和筛选内容。
- 编辑正文。
- 拆分、合并、恢复原文、删除。
- 手动新增内容。
- 查看原文对照和系统处理说明。
- 保存草稿。
- 生成检索索引。

交互规则：

- 运营编辑的是内容草稿，不直接编辑 embedding 或 chunk。
- 每次保存草稿后，索引状态变为待更新。
- 发布前必须生成检索索引。

### 5.7 清洗轮次记录

清洗轮次记录用于查看每一轮清洗产出的内容草稿和索引内容。

列表字段：

- 轮次。
- 本轮内容。
- 本轮清洗策略。
- 状态。
- 操作。

操作规则：

- 每一轮都支持 `查看`，打开抽屉查看当前清洗出的所有索引内容。
- 当前轮如已完成风险确认和内容确认，可进入检索试查或发布。
- 历史轮次只允许查看，不允许编辑。

### 5.8 检索试查

检索试查用于发布前验证内容是否能被搜到、召回是否符合预期。

页面能力：

- 输入用户问题。
- 选择针对当前草稿或当前线上内容试查。
- 查看相关结果、摘要和相似度。
- 内容修复场景自动带入待修复问题。
- 支持 `启动下一轮优化`，弹窗填写优化方向描述。
- 支持在无待确认项且索引已更新时发布。

### 5.9 发布结果

发布结果用于查看线上生效情况和历史版本。

展示内容：

- 线上内容名称。
- 发布时间。
- 发布人。
- 线上状态。
- 本次发布信息。
- 发布备注。
- 版本记录。

规则：

- 已发布任务的其他处理 Tab 不允许继续操作。
- 发布结果页保留查看版本内容和回滚入口。
- `查看当前线上内容` 和 `复制为新任务` 第一版隐藏。

## 7. 非目标

- 第一版不支持在线抓取 TXT 外链内容。
- 第一版不展示或配置自动化测试集。
- 第一版不展示测试覆盖率缺失 Tab。
- 第一版不把人工审核结论沉淀成全局规则。
- 第一版不让运营选择底层内部版本 ID。
- 第一版不让运营直接编辑 chunk、embedding 或向量索引。
- 第一版不直接覆盖历史发布版本。

## 9. 原型设计说明

### 9.1 页面入口

模块放在 cqil-studio：

```text
机器人管理
-> 知识库自动化
```

页面采用后台管理页结构。顶部只展示当前客户、知识库状态和当前线上版本状态。

### 9.2 首页

客户已有知识库时：

- 显示统计卡片。
- 显示维护任务列表。
- 显示当前线上版本和历史版本。
- 主按钮为 `新建维护任务`。

客户无知识库时：

- 隐藏统计卡片和任务列表。
- 展示空状态：`当前客户还没有知识库`。
- 主按钮为 `创建知识库`。

### 9.3 创建知识库

进入条件：

- 当前客户无知识库。

页面行为：

- 标题为 `创建知识库`。
- 所属客户只读。
- 显示新知识库名称。
- 不显示维护对象。
- 不显示任务类型。
- 第二步只支持上传第一版完整原文件。
- 创建后进入任务详情，看第一版清洗进度。

### 9.4 新建维护任务

进入条件：

- 当前客户已有知识库。

页面行为：

- 标题为 `新建维护任务`。
- 所属客户只读。
- 当前知识库只读。
- 不显示维护对象下拉。
- 显示任务类型。
- 人工补充显示多条手动录入列表。
- 内容修复同时支持上传原文件和手动新增问题。

### 9.5 任务详情

任务详情的一级 Tab：

- 处理进度与维护记录。
- 风险与确认。
- 清洗结果确认。
- 清洗轮次记录。
- 检索试查。
- 发布结果。

交互重点：

- 有风险项时，清洗结果确认不可进入。
- 风险处理完成后，进入清洗结果确认。
- 清洗结果编辑后必须重新生成检索索引。
- 检索试查不符合预期时，可启动下一轮优化。
- 发布后只读，默认跳到发布结果。

### 9.6 抽屉与弹窗

需要的抽屉或弹窗：

- 待确认内容抽屉：合并冲突确认、高风险内容删除、已确认通过、已忽略/删除内容。
- 版本内容抽屉：查看当前线上或历史发布版本的内容。
- 清洗轮次索引内容抽屉：查看某一轮清洗出的所有索引内容。
- 手动新增内容弹窗：在清洗结果确认中新增内容草稿。
- 下一轮优化弹窗：在检索试查中填写优化方向。
- 发布确认弹窗：确认未完成项、索引状态和发布备注。

### 9.7 视觉和文案原则

- 页面尽量使用运营能理解的语言，不暴露内部工程字段。
- 按“现在该做什么”组织操作按钮，少展示不必要的配置。
- 表格只承载概览，复杂判断用卡片、抽屉或弹窗展示。
- 危险操作必须二次确认。
- 已发布内容只读，避免误操作历史版本。
