# 会话质检异步创建设计

## 背景

当前“创建任务”接口会在一次请求中同步完成以下全部工作：

- 读取上传文件
- 解析知识库文件
- 解析历史对话 Excel
- 写入任务、知识块、会话和轮次
- 返回完整详情

当文件较大、文件较多，或某类文档解析较慢时，前端会一直停留在提交中的加载态。

## 目标

- 创建任务请求快速返回，不再阻塞在完整解析流程上
- 用户创建后立刻进入任务详情页
- 任务解析过程中有明确状态，避免“无限转圈”
- 解析完成后任务自动变为可运行

## 非目标

- 不引入外部消息队列或独立 worker 服务
- 不重写会话质检运行流程
- 不改变现有知识库切块与历史对话解析规则

## 方案

### 新增解析中状态

会话质检任务状态新增 `parsing`。

状态语义调整为：

- `parsing`: 上传完成，后台正在解析文件并写入任务数据
- `draft`: 解析完成，任务可开始检查
- `running`: 正在执行会话质检
- `completed`: 检查完成
- `failed`: 解析或检查失败

### 创建接口改为快速返回

`POST /api/projects/[id]/conversation-audit-jobs` 不再同步完成全量解析，而是：

1. 先创建任务记录，初始状态为 `parsing`
2. 把历史对话文件和知识库文件写入 job 级临时目录
3. 立即返回任务基础信息
4. 在后台异步启动解析流程

### 解析流程后台执行

新增独立的服务函数负责：

- 从 job 临时目录读取上传文件
- 复用现有文档解析与知识切块逻辑
- 解析历史对话 workbook
- 写入知识块、会话、轮次
- 更新 `parseSummary`
- 成功后把状态从 `parsing` 更新为 `draft`
- 失败时把状态更新为 `failed` 并写入 `errorMessage`
- 完成后清理 job 临时目录

### 前端创建后立即切换详情

前端创建成功后：

- 立即退出创建态并切换到新任务详情
- 当任务状态为 `parsing` 时，显示“正在解析上传文件”
- 禁用“开始检查”“导出 Excel”“只看问题”
- 轮询任务详情，直到状态不再是 `parsing`

### 详情接口兼容空结果

`GET /api/conversation-audit-jobs/[id]` 在 `parsing` 时允许返回空的 `conversations` 和 `turns`，由前端用任务状态区分“尚未解析完成”和“解析后没有数据”。

## 测试

- 创建接口在接收上传后立即返回 `parsing` 状态的任务
- 后台解析成功后，任务状态更新为 `draft`
- 后台解析失败后，任务状态更新为 `failed` 且有错误信息
- 详情页在 `parsing` 时显示解析中提示并禁用运行相关按钮
