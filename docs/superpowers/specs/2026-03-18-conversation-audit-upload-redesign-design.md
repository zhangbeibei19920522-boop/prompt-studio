# 会话质检上传区改版设计

## 背景

当前“新建会话质检任务”页面仍然使用两个原生文件输入：

- 历史对话 Excel
- 知识库文件

这带来两个问题：

1. 知识库上传虽然已经允许多文件，但入口不够直观，用户不容易理解可以一次混合上传多种格式。
2. 两个上传按钮视觉权重太低，页面的主要动作不明显。

## 目标

- 把两个上传入口都改成清晰的拖拽上传区
- 明确表达知识库支持一次混合上传 `Word + HTML + Excel`
- 保持现有创建任务接口和后端解析逻辑不变

## 非目标

- 不改动会话质检的数据模型
- 不改动创建任务 API
- 不改动知识库解析、切轮、评估和导出逻辑

## 方案

### 上传区结构

创建任务页保留两个独立上传入口，均改为卡片式拖拽区：

- `历史对话 Excel`
- `知识库文件`

两个卡片都支持：

- 点击卡片打开文件选择器
- 拖拽文件到卡片进行上传
- 拖拽悬停时高亮边框和背景

### 文件约束

历史对话卡片：

- 只接受 `xls/xlsx`
- 单文件模式
- 新选择的文件覆盖旧文件

知识库卡片：

- 接受 `doc/docx/html/htm/xls/xlsx`
- 多文件模式
- 允许一次混合上传多种格式文件

### 视觉表现

每个卡片都应突出“上传是主操作”：

- 大尺寸虚线边框区域
- 上传图标
- 主提示文案“点击上传或拖拽文件到此处”
- 支持格式说明
- 选中文件后展示文件名或文件列表

知识库卡片额外强调：

- 支持混合上传多种格式

历史对话卡片额外强调：

- 仅支持 Excel
- 将按 `Conversation ID` 解析会话

## 交互细节

- 知识库文件列表显示已选文件数量和文件名
- 历史对话文件显示当前已选单个文件名
- 保留现有“创建任务”按钮作为提交动作
- 当历史对话文件缺失或任务名称为空时，创建按钮仍保持禁用

## 实现边界

主要修改文件：

- `src/components/audit/conversation-audit-detail.tsx`

必要时可在同目录或 `src/components/ui` 下增加小型复用上传区组件，但不做无关抽象。

## 验证

- 创建页能通过点击和拖拽两种方式选择文件
- 知识库上传区可一次选入混合格式文件
- 历史对话上传区拒绝非 Excel 文件
- 原有创建任务流程保持可用
