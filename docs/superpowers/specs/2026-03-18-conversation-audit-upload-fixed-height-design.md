# 会话质检上传区固定高度设计

## 背景

当前“新建会话质检任务”页的上传区已经改成拖拽卡片，但已选文件列表会随文件数量持续增高。

结果是：

- 知识库文件很多时，上传区整体被撑高
- “创建任务”按钮被挤出当前可视区域
- 页面主要提交动作不稳定

## 目标

- 上传很多文件时，创建页布局保持稳定
- 两个上传框都使用固定高度
- 已选文件很多时，仅在上传框内部滚动
- “创建任务”按钮始终留在上传区下方可见

## 非目标

- 不改动上传格式限制
- 不改动任务创建接口
- 不改动拖拽上传行为

## 方案

### 固定高度容器

两个上传框都改为固定高度卡片容器，内部拆成两个区域：

- 上部：拖拽上传区
- 下部：已选文件区

容器总高度固定，避免随着文件数量变化而继续向下增长。

### 文件列表区域

已选文件区域始终占位，不再根据是否有文件决定整体高度。

- 无文件时显示空态提示
- 有文件时显示已选数量和文件名列表
- 文件过多时只在该区域内部滚动

### 布局稳定性

- 历史对话上传框和知识库上传框保持相同结构
- 错误提示保留在卡片底部
- 创建按钮位置不再受文件数量影响

## 验证

- 上传大量知识库文件后，页面整体高度不再持续增长
- 创建按钮仍留在上传区下方
- 文件列表可在卡片内部滚动
- 空态和有文件态切换时，上传框整体高度保持稳定
