# Conversational Workspace Redesign Design

## Goal

将当前零散的多模块主界面重组为一个以对话为入口、以工作台为主区的统一工作区：

- 对话保持唯一入口
- `Prompt`、`测试`、`会话质检` 成为三种平级重工作台
- 知识库降为按需打开的抽屉
- 统一使用命令面板召回资产和执行动作

## Current Problems

- 左侧栏同时承担会话、Prompt、文档、测试、质检等多种导航职责，信息过载
- 对话、测试、质检三种主场景在同一页里直接切换，但没有统一工作区结构
- Prompt 主要通过右侧窄面板操作，不适合成为主工作台
- 知识库长期占位，挤压主线程和主任务的注意力

## Approved Direction

- 整体感觉采用 `Cursor/IDE + 对话驱动工作区`
- 三类重工作台：`Prompt`、`测试`、`会话质检`
- 知识库作为抽屉
- 左侧只保留会话线程
- 命令面板为混合型：既能搜索资产，也能直接执行动作
- 资产与对话为弱绑定：可以由对话发起，但之后可独立召回

## Interaction Model

### 1. Workspace Shell

- 顶部：项目切换、命令面板入口、知识库抽屉入口、全局设置
- 左侧：当前项目下的会话线程
- 中央：对话主线程 / Prompt 工作台 / 测试工作台 / 质检工作台

### 2. Conversational Routing

- 默认落点始终为对话主线程
- Agent 在对话中输出动作建议和路由上下文
- 打开工作台后，不跳转到独立页面；而是在主区切换到对应工作台
- 左侧线程保持可见，用户可以随时回到当前对话

### 3. Asset Recall

- 命令面板负责：
  - 新建 Prompt
  - 新建测试集
  - 新建质检任务
  - 搜索并打开已有 Prompt / 测试 / 质检 / 文档
- 文档与知识库文件通过抽屉呈现，并可直接投喂给当前对话

## Visual Direction

- 摒弃旧的纯白后台三栏工具感
- 使用暖中性色背景 + 深色标题区 + 更强的卡片层级
- 主工作区保持高信息密度，但避免传统后台表格式堆叠
- 对话、工作台、命令面板、抽屉在视觉上属于同一系统

## Implementation Boundaries

- 第一阶段只重构主工作区壳层与导航结构
- 聊天、测试、质检的深层业务逻辑继续复用现有组件
- Prompt 视图从右侧窄面板迁移到中央工作区
- 旧 `TopBar` / `Sidebar` / `RightPanel` 不再作为主结构使用

## Testing

- 为新工作区壳层补渲染测试，锁住：
  - 命令面板入口
  - 知识库抽屉入口
  - 左侧线程栏
  - 三类工作台切换入口
- 跑页面相关测试和最小构建检查，确保新壳层没有打断现有功能挂载
