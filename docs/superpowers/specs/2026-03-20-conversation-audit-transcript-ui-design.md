# 会话质检对话内容 UI 对齐测试模块设计

## 目标

将会话质检详情中的“对话内容”区域改成与测试模块一致的对话列表样式，但不展示 `intent`、路由结果等测试模块专属信息。

## 设计

- 保留会话质检详情页现有的卡片头部、展开折叠逻辑、“流程规范检查”和“知识问答问题”区域。
- 仅替换展开态中的“对话内容”渲染方式，复用测试模块的 `ConversationPanel`。
- 将每个 `ConversationAuditTurn` 转换为顺序消息列表：
  - `userMessage` -> `user`
  - `botReply || "无回复"` -> `assistant`
- 视觉上使用测试模块已有的“用户 / 助手”标签和分段列表容器，形成 `用户：xxx`、`助手：xxx` 的阅读效果。
- 不传入 `intent` 数据，也不显示 intent badge。

## 验证

- 更新会话质检详情测试，断言：
  - “对话内容”使用测试模块同款列表容器样式。
  - 渲染“用户”“助手”标签。
  - 缺失回复时展示“无回复”。
  - 不再出现旧的“机器人”文案。
