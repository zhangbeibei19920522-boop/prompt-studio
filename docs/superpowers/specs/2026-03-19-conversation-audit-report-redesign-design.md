# 会话质检报告式结果页改版设计

## 背景

当前会话质检模块存在三个主要问题：

- 任务不支持删除
- 结果页以逐轮卡片为主，信息密度低，可读性差
- 评估逻辑只覆盖单轮问答，无法判断整段会话是否符合知识库中的流程规范

同时，用户希望最终交付更接近产品化结果页，而不是开发视角的调试面板：

- 第一屏直接看到对话总数和整体通过率
- 同时区分“流程是否正确”和“知识问答是否正确”
- 能快速查看每个对话的评估情况
- 对于流程异常，明确指出错在哪个环节
- 对于知识问答错误，给出正确回答和来源文件

## 目标

- 将结果页改为“报告首页 + 对话评估清单 + 对话详情”的报告式工作区
- 将评估对象从“逐轮问答”扩展为“会话级流程评估 + 轮次级知识问答评估”
- 在同一个任务里同时输出流程结果和知识问答结果
- 支持删除会话质检任务

## 非目标

- 不引入单独的流程模板配置中心
- 不要求用户额外上传流程文件
- 不做复杂的可视化图表系统
- 不改变历史对话上传格式

## 交互结构

### 1. 报告首页

首页只保留核心指标和风险摘要，不再铺满逐轮结果。

展示内容：

- 对话总数
- 整体通过率
- 流程通过率
- 知识问答通过率
- 高风险对话数或异常对话数

整体通过率按“对话”计算，不按轮次计算：

- 一段对话只要流程不通过，或存在知识问答错误，则该对话记为未通过

### 2. 对话评估清单

主列表按对话展示，每个对话一条记录，不再按轮次堆卡片。

每条记录至少展示：

- 对话 ID
- 整体结果：通过 / 未通过
- 流程结果：通过 / 异常 / 无法判断
- 知识结果：通过 / 异常 / 无相关问答
- 问题摘要

列表需要支持：

- 按全部 / 未通过 / 流程异常 / 知识错误筛选
- 按风险优先排序

### 3. 对话详情

点击某条对话后，展示两个正式模块：

#### 流程规范检查

按整段会话判断是否符合知识库中的流程要求，输出：

- 流程是否通过
- 流程步骤列表
- 每个步骤的状态：已完成 / 缺失 / 顺序异常 / 内容偏差 / 不适用
- 每个异常步骤的说明
- 关联轮次证据
- 关联来源文件

#### 知识问答问题

只展示存在问题的轮次，输出：

- 第几轮
- 用户问题
- 机器人回答
- 建议回答或正确回答
- 来源文件

## 评估模型

### 双层评估

同一个任务内同时运行两套结果产出：

#### 1. 会话级流程评估

输入：

- 整段会话全文
- 从知识库中召回的流程相关内容

输出：

- 流程是否通过
- 流程步骤清单
- 每一步的判定结果
- 异常摘要

#### 2. 轮次级知识问答评估

保留现有逐轮评估思路，但结果只在详情页中作为“知识问答问题”模块展示。

输出：

- 当前轮次是否有问题
- 建议回答 / 正确回答
- 来源文件

### 任务级汇总

任务层需要从会话结果反推汇总指标：

- 对话总数
- 整体通过对话数
- 流程通过对话数
- 知识通过对话数
- 高风险对话数

## 数据与结果结构

现有结构只有：

- 任务
- 会话
- 轮次

改版后需要补充“会话级评估结果”，而不是继续把流程结果硬塞进单轮记录。

建议新增或扩展会话级字段，用于存储：

- 整体结果
- 流程结果
- 知识结果
- 风险等级
- 问题摘要
- 流程步骤明细

轮次级记录继续保留知识问答相关信息，用于细节展开。

## 删除任务

任务删除应为正式产品能力，放在任务级操作中。

行为要求：

- 删除前二次确认
- 删除后同时移除任务、会话、轮次、知识块及关联结果
- 删除成功后列表立即刷新
- 当前正在查看该任务时，删除后回到任务列表默认状态

## 导出

导出不应只导出逐轮表格。

建议导出至少包含两个 sheet：

- `Summary`：任务级指标和对话级结果摘要
- `Conversation Details`：每个对话的流程结果、知识问题、建议回答、来源文件

如果保留逐轮 sheet，可作为附加明细，而不是唯一结果。

## 组件方向

结果页需要明显产品化，避免说明式占位内容和过度铺陈。

视觉方向：

- 首页更简洁，突出核心指标和异常摘要
- 对话清单保持紧凑，不用大段卡片文案
- 详情页两栏或上下结构清晰分离“流程”和“知识问答”
- 默认只展示用户真正会看的内容，次级信息通过展开或标签承载

## 验证

- 首页可直接看到对话总数和整体通过率
- 可以快速查看每个对话的整体、流程、知识三类结果
- 对于流程异常，可看到具体异常步骤和环节说明
- 对于知识问答错误，可看到正确回答和来源文件
- 删除任务后，相关数据被完整移除，页面状态正确更新
