# 会话质检知识库文件叠加上传设计

## 背景

当前会话质检创建页里的“知识库文件”上传卡，在用户第二次选择文件时会直接覆盖第一次已选中的文件。

这与多文件上传的预期不一致。用户需要分多次补充上传，未重名的文件应该继续保留；如果新上传文件与已有文件重名，则用新文件替换旧文件。

## 目标

- 知识库文件支持分多次叠加选择
- 未重名文件保留并追加
- 重名文件使用新文件替换旧文件

## 非目标

- 不改历史对话 Excel 的单文件上传行为
- 不改后端 `FormData` 提交格式
- 不改已有文件类型过滤规则

## 方案

### 知识库文件按名称增量合并

“知识库文件”上传卡在 `multiple` 模式下，不再用新一批文件直接替换当前状态，而是把“当前已选文件”和“本次新选文件”做一次按文件名的合并。

合并规则：

- 文件名未出现过：直接追加
- 文件名已存在：保留新文件，替换旧文件

### 单文件上传保持原行为

“历史对话 Excel”继续沿用单文件模式，每次新选择都覆盖旧值。

## 测试

- 第一次选择 `a.docx` 后，再选择 `b.html`，列表应同时显示两个文件
- 已有 `faq.docx` 时，再选择同名 `faq.docx`，列表只保留一份
- 非法扩展名仍被忽略，错误提示保持现状
